# 工作汇报与会议纪要格式模板使用说明

## 新增模板概览

本次更新为 Word 格式化工具新增了两种常用的职场文档格式模板：

### 1. 工作汇报格式 (`work_report.json`)
适用于：年终总结、述职报告、项目总结等详细工作汇报

### 2. 会议纪要格式 (`meeting_minutes.json`)
适用于：部门例会、项目评审会、周/月例会等会议记录

---

## 工作汇报格式规范

### 页面设置
- **纸张大小**: A4 (210mm × 297mm)
- **页边距**: 上下 2.54cm，左右 3.18cm（Word 默认边距）
- **装订线**: 左侧 0.5cm（如需打印装订）

### 字体与字号

| 元素 | 中文字体 | 英文字体 | 字号 | 样式 |
|------|----------|----------|------|------|
| 大标题 | 方正小标宋简体 | Arial Black | 二号 (22pt) | 加粗，居中 |
| 一级标题 | 黑体 | Arial | 三号 (16pt) | 加粗，左对齐 |
| 二级标题 | 楷体_GB2312 | Times New Roman | 三号 (16pt) | 加粗，左对齐 |
| 三级标题 | 仿宋_GB2312 | Times New Roman | 三号 (16pt) | 加粗，左对齐 |
| 正文 | 仿宋_GB2312 | Times New Roman | 三号 (16pt) | 常规，两端对齐 |

### 段落与间距
- **行距**: 固定值 28 磅（公文标准）
- **首行缩进**: 正文段落首行缩进 2 字符
- **对齐方式**: 正文两端对齐
- **段间距**: 段后 0.5 行（不使用连续回车空行）

### 列表编号规范
1. **第一层级**: 一、二、三、（中文顿号）
2. **第二层级**: （一）（二）（三）（中文括号，括号外不加标点）
3. **第三层级**: 1. 2. 3.（阿拉伯数字加下脚点）
4. **第四层级**: （1）（2）（3）

### 表格规范
- **表头**: 加粗，浅灰色底色 (RGB: 217, 217, 217)
- **表格内容**: 居中或左对齐，字号比正文小一号
- **边框**: 细实线，推荐"三线表"样式

### 页眉页脚
- **页眉**: "202X年度工作汇报"，小五号字体
- **页脚**: "第 X 页 / 共 Y 页"，居中

## 会议纪要格式规范

### 页面设置
- **纸张大小**: A4
- **页边距**: 上下 2.54cm，左右 3.18cm（标准）

### 字体与字号

| 元素 | 中文字体 | 英文字体 | 字号 | 样式 |
|------|----------|----------|------|------|
| 文档标题 | 方正小标宋简体 | Arial | 二号 (22pt) | 加粗，居中 |
| 会议信息头 | 仿宋_GB2312 | Times New Roman | 四号 (14pt) | 常规 |
| 一级标题 | 黑体 | Arial | 三号 (16pt) | 加粗，左对齐 |
| 二级标题 | 楷体_GB2312 | Times New Roman | 三号 (16pt) | 加粗，左对齐 |
| 正文内容 | 仿宋_GB2312 | Times New Roman | 四号 (14pt) | 常规 |
| 待办事项 | 微软雅黑 | Arial | 四号 (14pt) | 加粗/高亮 |

### 结构与排版

#### 会议基本信息区
采用"标签：内容"的列表形式，确保对齐：
```
会议主题：2024年Q3产品规划评审会
会议时间：2024-07-15 14:00 - 16:00
会议地点：301会议室 / 腾讯会议
主持人：张三
记录人：李四
参会人员：王五、赵六、孙七（请假：周八）
```

#### 正文层级逻辑
1. **会议决议**（最重要，放在最前面，结论先行）
2. **主要讨论内容**（按议题分类，不按发言顺序）
3. **待办事项**（必须使用表格）

#### 待办事项表格规范
| 列名 | 说明 |
|------|------|
| 序号 | 任务编号 |
| 任务内容 | 详细描述 |
| 责任人 | 执行人姓名 |
| 截止日期 | 完成时间 |
| 备注 | 补充说明 |

- **表头**: 浅灰底色，加粗，居中
- **内容**: 左对齐（长文本）或居中（短文本/日期）
- **边框**: 仅保留横线（三线表风格）
- **高亮**: 逾期或紧急任务可用浅红色背景

### 页眉页脚
- **页眉**: 左侧"XX公司会议纪要"，右侧"内部资料·注意保密"
- **页脚**: "第 X 页 / 共 Y 页"，居中

## 使用方法

### 1. 启动应用
```bash
cd D:\study\word-format-tool
streamlit run app.py
```

### 2. 上传文档
在左侧边栏上传需要格式化的 `.docx` 文件。

### 3. 选择模板
在"选择格式化模板"下拉菜单中选择：
- **工作汇报格式** - 用于年终总结、述职报告等
- **会议纪要格式** - 用于会议记录、待办事项等

### 4. 预览与修正
系统会自动识别文档中的标题层级，你可以在表格中手动修正：
- `title` - 文档总标题
- `heading_1` - 一级标题（一、二、三、）
- `heading_2` - 二级标题（（一）（二）（三））
- `heading_3` - 三级标题（1. 2. 3.）
- `heading_4` - 四级标题（（1）（2）（3））
- `body` - 正文内容

### 5. 应用格式化
点击"🚀 开始格式化"按钮，系统会自动应用模板格式。

### 6. 下载结果
格式化完成后，点击"📥 下载格式化后的文档"保存结果。

## 注意事项

1. **字体安装**: 确保系统已安装以下字体：
   - 方正小标宋简体
   - 黑体
   - 楷体_GB2312
   - 仿宋_GB2312
   - 微软雅黑

2. **中英文混排**: 模板已配置中英文字体分离，中文使用指定中文字体，英文/数字使用 Times New Roman 或 Arial。

3. **行距设置**: 使用固定值行距（工作汇报 28 磅，会议纪要 26 磅），确保排版一致性。

4. **表格处理**: 会议纪要中的待办事项必须使用表格，系统会自动应用表格样式。

5. **云环境限制**: 在 Streamlit Cloud 等云环境下，模板无法持久化保存，需每次重新上传。

## 常见问题

**Q: 为什么格式化后字体没有变化？**
A: 检查系统是否安装了模板中指定的字体。可以在"系统字体环境检测"中查看可用字体。

**Q: 如何自定义模板？**
A: 点击侧边栏"🛠️ 模板管理"进入模板编辑器，可以可视化修改字体、字号、行距等参数。

**Q: 会议纪要的待办事项表格如何高亮紧急任务？**
A: 目前模板支持表头浅灰底色，手动高亮需要在 Word 中打开后设置。后续版本将支持自动高亮。

**Q: 工作汇报的列表编号能自动生成吗？**
A: 系统会智能识别中文编号格式（一、（一）、1.等），建议在 Word 中使用多级列表功能。
