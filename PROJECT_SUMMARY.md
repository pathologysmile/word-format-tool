# Word 文档格式自动化工具项目总结

## 1. 项目概述
本项目是一个基于 Streamlit 和 python-docx 构建的 Web 应用，旨在自动化处理 Word 文档的排版工作。它能够智能识别文档结构（标题、正文等），并根据预设或自定义的模板（如公文标准、学术论文、商务报告）一键完成复杂的格式化操作，显著提升了文档处理的效率与规范性。

**核心目标：** 解决手动调整 Word 格式繁琐、易出错的问题，实现“上传即规范”的智能化办公体验。

## 2. 技术栈
- **前端框架:** Streamlit (提供交互式 Web 界面)
- **核心引擎:** python-docx (Word 文档解析与生成)
- **数据处理:** Pandas (用于表格化展示与编辑段落层级)
- **底层支持:** lxml (XML 处理，确保行距等复杂格式的精确控制)
- **字体管理:** 跨平台字体检测工具 (支持 Windows/macOS/Linux)

## 3. 核心功能模块

### 3.1 智能标题识别 (`src/title_detector.py`)
采用“样式优先 + 正则补充 + 启发式推断”的混合识别策略：
- **样式匹配:** 识别 Word 内置的 Heading 1-4 及 Title 样式。
- **正则模式:** 针对中文序号（如“一、”、“(一)”、“1.”）进行高精度匹配。
- **人工修正:** 在 UI 界面提供可编辑的数据表格，允许用户在格式化前微调段落层级。

### 3.2 深度格式化引擎 (`src/document_formatter.py`)
- **全要素配置:** 支持页边距、纸张大小、页眉页脚、首行缩进、段前段后间距等。
- **精准行距控制:** 通过直接操作 XML 节点 (`w:spacing`)，实现了与 Microsoft Word 完全一致的固定值、多倍行距等设置，解决了 `python-docx` 原生 API 在某些场景下失效的问题。
- **中英文字体分离:** 能够根据字符类型（中文/数字/英文）自动应用不同的字体（如中文用仿宋，数字用 Times New Roman）。

### 3.3 可视化模板管理 (`app.py`)
- **预设模板:** 内置符合 GB/T 9704-2012 标准的公文模板，以及学术、商务常用模板。
- **在线编辑器:** 提供直观的 UI 界面，用户无需编写 JSON 即可实时调整各级标题的字号、颜色、间距并保存为新模板。
- **导入导出:** 支持 JSON 格式模板文件的分享与迁移。

### 3.4 云环境适配 (`src/font_utils.py`)
- **环境感知:** 自动检测是否运行在 Streamlit Cloud 等无字体环境中。
- **降级策略:** 在缺失特定中文字体时，自动映射到系统通用字体，确保程序不崩溃。
- **内存流处理:** 全程使用 `BytesIO` 处理文件，避免在云服务器上产生临时文件权限问题。

## 4. 项目架构与目录
```text
word-format-tool/
├── app.py                  # Streamlit 主入口与 UI 逻辑
├── src/                    # 核心业务逻辑层
│   ├── document_parser.py  # 文档内容提取与元数据分析
│   ├── title_detector.py   # 标题层级智能识别算法
│   ├── document_formatter.py # 格式化执行引擎
│   └── font_utils.py       # 跨平台字体检测与管理
├── templates/              # JSON 格式化模板存储目录
├── tests/                  # 单元测试与集成测试脚本
├── font/                   # 备用字体文件
└── requirements.txt        # 依赖清单
```

## 5. 关键亮点
1. **GB/T 9704-2012 标准支持:** 深度还原了党政机关公文的严格排版要求。
2. **所见即所得的交互:** 用户在点击“开始格式化”前，可以通过预览界面确认并修改 AI 识别出的文档结构。
3. **高鲁棒性:** 针对云端部署和本地运行进行了双重优化，具备良好的跨平台兼容性。

## 6. 总结
该项目成功将复杂的 Word 自动化排版逻辑封装为简单易用的 Web 工具。通过模块化设计，不仅满足了当前的公文处理需求，也为后续扩展更多行业排版标准（如医疗病历、法律文书）奠定了坚实的基础。
